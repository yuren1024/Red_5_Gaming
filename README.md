# 红五五家记分工具

一个基于浏览器的红五五家卡牌游戏记分和统计工具，支持本地存储，无需服务器。

## 📋 项目介绍

红五五家记分工具是一个专为5人局红五游戏设计的客户端记分应用。该工具按照百度百科规则实现，支持一般情况、独打和逃跑三种结算模式，能够自动计算得分、累计总分、生成排行榜，并支持数据导出。

### 核心功能

- **多模式记分**：支持一般情况、独打、逃跑三种游戏模式
- **自动结算**：根据闲家抓分自动计算各方得分
- **玩家管理**：固定5个玩家槽位，支持自定义改名
- **数据统计**：实时计算总分、胜率、叫庄率
- **排行榜**：按总分排序展示玩家排名
- **历史记录**：完整的对局记录，支持编辑和删除
- **数据导出**：支持CSV和JSON格式导出
- **本地存储**：数据保存在浏览器本地，无需联网

## 🎮 使用指南

### 登录系统

1. 打开应用后会自动跳转到登录页面
2. 使用默认账号登录：
   - 账号：`于人`
   - 密码：`123456`
3. 登录成功后进入游戏选择页面

### 基本使用流程

1. **进入记分工具**
   - 在游戏选择页面点击"红五五家"卡片
   - 进入主记分界面

2. **设置玩家**
   - 在"玩家设置"区域修改5位玩家的姓名
   - 可随时恢复默认姓名（A-E）

3. **记录对局**
   - 填写局号（自动递增）
   - 选择游戏模式：一般情况/独打
   - 选择结束方式：抓分结算/逃跑
   - 选择庄家（独打模式无需选择伴家）
   - 选择伴家（一般情况模式）
   - 输入闲家抓分（抓分结算模式）或选择逃跑玩家（逃跑模式）
   - 点击"预览结算"查看得分预览
   - 确认无误后点击"保存本局"

4. **查看统计**
   - 数据看板显示总局数、当前领先玩家等信息
   - 排行榜展示所有玩家的总分、胜率、叫庄率
   - 历史记录表格显示每局详细信息

5. **数据管理**
   - 点击历史记录中的"编辑"按钮修改对局
   - 点击"删除"按钮删除对局记录
   - 使用"导出CSV"或"导出JSON"按钮备份数据

## 🛠️ 技术实现

### 技术栈

- **前端框架**：原生JavaScript + HTML5 + CSS3
- **数据存储**：浏览器 localStorage
- **认证方式**：简单的本地验证
- **部署方式**：静态文件，无需服务器

### 核心架构

```
Red_5_Gaming/
├── index.html          # 主记分界面
├── login.html          # 登录页面
├── games.html          # 游戏选择页面
├── app.js             # 核心业务逻辑
├── auth.js            # 认证模块
├── styles.css         # 样式文件
└── README.md          # 项目说明
```

### 实现原理

1. **状态管理**
   - 使用单一状态对象管理应用数据
   - 所有状态变更自动保存到localStorage
   - 页面刷新后自动恢复上次状态

2. **记分规则**
   - 内置两张规则表：一般情况规则表和独打规则表
   - 根据闲家抓分查找对应得分规则
   - 120分为胜负分界线

3. **数据流程**
   ```
   用户输入 → 表单验证 → 规则查找 → 得分计算 → 状态更新 → 界面渲染 → 本地存储
   ```

4. **统计计算**
   - 遍历所有对局记录累计每个玩家的总分
   - 根据胜负记录计算胜率
   - 根据庄家次数计算叫庄率

## 📊 规则说明

### 一般情况
- 庄家 + 伴家 vs 3名闲家
- 闲家抓分 < 120：庄家方胜
- 闲家抓分 ≥ 120：闲家方胜

### 独打
- 庄家 vs 4名闲家
- 闲家抓分 < 120：庄家胜
- 闲家抓分 ≥ 120：闲家胜

### 逃跑
- 一般模式：逃跑者扣80分，其余各加10分
- 独打模式：逃跑者扣200分，其余各加25分

## 🔒 数据安全

- 所有数据存储在用户浏览器本地
- 不会上传到任何服务器
- 清除浏览器数据会丢失所有记录
- 建议定期使用导出功能备份数据

## 📝 注意事项

1. 局号必须唯一，重复局号无法保存
2. 闲家抓分必须是5的倍数，范围0-300
3. 庄家和伴家不能是同一位玩家
4. 修改历史记录会自动重新计算所有统计数据
5. 建议在游戏过程中及时记录，避免遗忘
